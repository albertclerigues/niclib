class niclib2.net.train.Trainer(max_epochs, loss_func, optimizer, optimizer_opts=None, train_metrics=None, val_metrics=None, plugins=None, device='cuda')[source]¶

Bases: object

Class for torch Module training using a training and validation set with basic metric supprt. It also has support for plugins, a way to embed functionality in the training procedure.

Plugins are made by inheriting from the base class TrainerPlugin and overriding the inherited functions to implement the desired functionality.

Parameters

  • max_epochs (int) – maximum number of epochs to train. Training can be interrupted by setting the attribute keep_training to False.

  • loss_func (torch.nn.Module) – loss function.

  • optimizer – torch.optim.Optimizer derived optimizer.

  • train_metrics (dict) – dictionary with the desired metrics as key value pairs specifiying the metric name and the function object (callable) respectively. The loss function is automatically included under the key ‘loss’.

  • val_metrics (dict) – same as train_metrics.

  • plugins (list) – List of plugin objects.

  • device (torch.device) – torch device i.e. ‘cpu’, ‘cuda’, ‘cuda:0’…

Runtime variables accesible to the plugins via the input argument trainer.

  • keep_training (bool) – control flag to continue or interrupt the training. Checked at the start of each epoch.

class niclib2.net.train.EarlyStopping(metric_name, patience, min_delta=0.0001)[source]¶

Bases: niclib2.net.train.TrainerPlugin

Plugin to interrupt the training with the early stopping technique.

Parameters

  • filepath – Filepath where the best model will be stored.

  • metric_name (str) – monitored metric name. The name has to exist as a key in the val_metrics provided to the Trainer.

  • patience (int) – Number of epochs to wait since the last best model before interrupting training.

  • min_delta – minimum change between metric values to consider a new best model.
